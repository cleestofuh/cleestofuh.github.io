Whova, Attendance Manager

Whova


Whova is an event management system where event organizers can manage their agenda, attendees, as well as promote networking within their event. This system includes a mobile event app and a web based dashboard where event organizers can configure and set up their event information on the app.

Whova provides mobile event solutions to help make event planning easier and to help improve the way attendees navigate through their event.

Winner of the People's Choice Award at the Event Technology Awards 2018 →

Overview


We wanted to design a solution that helps event organizers better manage attendance of their agenda sessions and gives them more confidence in their planning.

My role was to lead the design of a new feature that would integrate with the event organizer dashboard as well as the app. This case study will focus on the organizer dashboard.

The Problem


To start off, I spoke to a few of our past customers to understand their needs and pain points during their agenda planning. Based on user interviews, there were 2 main challenges that organizers faced:

  • Making sure their sessions were not overbooked due to physical room capacities.
  • They needed a way to gauge interest in each session before the event so they could adjust their agenda accordingly by either swapping rooms or adding more sessions.

Design Iteration



First design iteration

After some competetive analysis and brainstorming, I created a prototype to get a sense of how the new feature flow would work. This included adding an "Attendance" tab within the session editor to be able to set a capacity, enable a waitlist, and also enable the ability to check-in attendees.


When testing the flow, there was one major problem that came up: Visibility


For the majority of our users, they upload and edit their event agenda through an excel spreadsheet. This led to a low visibility of the feature to begin with since to use it, they would need to be editing individual session through the dashboard, which is not the method that most users go through.

Another insight that was expressed was that in many cases, if the event had a session that needed a capacity set, or a session check-in feature, these would need to be enabled for ALL their sessions. From the current flow, the user would have to tediously go through each session one by one.

Revisions



Initial Release

From the feedback, we knew the main issues that we wanted to address for the first release. To address the visisbility issue, we decided to separate this feature into it's own page within the dashboard. This resulted in a more focused experience where the session attendance management is the main task on this page. We also added an option to add all the sessions at once to reduce time needed to set up all of the session capacitites.

Next Steps


For this release, we wanted to get the core functionality out for our users in time for upcoming events. In the future, we will be tackling more use cases that require further research and thinking. A few of these include:

  • Overriding of session capacities for VIP attendees
  • More options for waitlisted attendees (give priority, move up in the waitlist, etc)
  • Linking our registration system into capped sessions so attendees can sign up for sessions when purchasing their ticket
